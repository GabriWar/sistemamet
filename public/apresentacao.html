Pilar 1 — Tema & Justificativa

Por que recomendar com grafos?

Tema. Um explorador de obras — filmes, séries, livros e animes — que conecta títulos por semelhança de conteúdo e deixa o usuário navegar essas conexões.

O problema. Recomendadores comuns são caixas-pretas: dizem "você vai gostar disto" sem dizer por quê.

Justificativa. Modelar o acervo como um grafo torna a recomendação transparente e auditável: cada conexão é uma conta que dá pra abrir, e a navegação vira um problema clássico de grafos (vizinhança, caminhos, distância).

Pilar 2 — Modelagem computacional

Da obra ao nó: atributos e similaridade

Cada obra é descrita por atributos em 4 dimensões:

  • Gêneros — fantasia, ficção-científica, crime…
  • Temas — poder, vingança, identidade…
  • Tom — sombrio, épico, melancólico…
  • Ambientação — medieval, futuro, cidade…

A similaridade entre duas obras é a sobreposição de atributos por dimensão, ponderada por peso — usando o coeficiente de sobreposição:

overlap(A,B) = |A ∩ B| / min(|A|,|B|)
sim = Σ dim pesodim · overlapdim
gêneros 0,30 · temas 0,30 · tom 0,20 · ambientação 0,20
Pilar 2 — Modelagem computacional

O acervo vira um grafo ponderado

  • Vértices (V): as ~290 obras.
  • Arestas (E): par de obras com sim ≥ 0,45 (limiar).
  • Peso: a própria similaridade (0–1) na aresta.
  • Não-direcionado — semelhança é simétrica.
  • Construído por varredura de todos os pares O(V²).
// para cada par de obras, cria aresta se passar do limiar
for i < n:
  for j > i:
    s = similarity(W[i], W[j])
    if s.score >= 0.45:
      links.push({ source, target,
                  score: s.score,
                  breakdown: s.breakdown })

Cada aresta guarda o breakdown — quanto cada dimensão contribuiu — o que torna a recomendação explicável.

Pilar 3 — Estruturas de dados

Como o grafo é guardado

1Lista de adjacência

Map<id, [{id, score}]> — vizinhos de cada nó. Base dos algoritmos de caminho. Memoizada (construída uma vez).

2Lista de arestas

[{source,target,score,breakdown}] — usada para desenhar e para o "por quê".

3Hash / Map

WORKS_BY_ID — acesso O(1) à obra por id. Catálogo como lista de objetos.

4Set (conjuntos)

Atributos por dimensão; visitados no Dijkstra; nós da pool; deduplicação de subgrafos.

5Dicionário de co-ocorrência

cooc["a|b"] = contagem — frequência de pares de conceitos (grafo de conceitos).

6Subgrafos sob demanda

Ego, pool e rotas geram grafos parciais — só o necessário é materializado.

Pilar 4 — Algoritmos de grafos

Quais algoritmos rodam por baixo

  • Construção por similaridade — all-pairs O(V²).
  • Subgrafo ego — vizinhança top-K de um nó (expansão).
  • Dijkstra — caminho de maior afinidade entre 2 obras.
  • Yen (K-shortest paths) — 3 rotas distintas.
  • Co-ocorrência + poda top-K — grafo de conceitos.
  • Ranking por média / argmin — pool e obra mais distante.
  • Force-directed layout — posicionamento (d3-force).
Pilar 4 — Algoritmo em foco

Dijkstra — o caminho mais "forte"

Para conectar duas obras, buscamos o caminho que passa pelas ligações mais fortes. Como Dijkstra minimiza custo, definimos o custo de cada aresta como o inverso da similaridade:

custo(aresta) = 1 − similaridade (+ ε)

Assim, somar custos = preferir caminhos com elos mais parecidos. Complexidade O(V²) nesta implementação.

function dijkstra(src, dst):
  dist[src] = 0;  visited = Set()
  while existe não-visitado:
    u = nó de menor dist
    if u == dst: break
    visited.add(u)
    for (v, score) in adj[u]:
      nd = dist[u] + (1 - score)
      if nd < dist[v]: dist[v]=nd; prev[v]=u
  return reconstruir(prev, dst)
Pilar 4 — Algoritmo em foco

Yen — três rotas: curta, média e longa

O algoritmo de Yen acha os K menores caminhos sem repetir nó, reaplicando Dijkstra a partir de "nós de desvio" (spur). Geramos K=18 candidatos e escolhemos 3 por comprimento: a mais curta, uma intermediária e a mais longa.

mais curta média mais longa

Ex.: Batman → Better Call Saul → Coringa → Taxi Driver. Cada salto mostra o que as duas obras compartilham.

// Yen, em alto nível
A = [ dijkstra(a, b) ]
for k in 1..K-1:
  for cada nó-desvio do caminho A[k-1]:
    remove arestas/nós já usados
    spur = dijkstra(desvio, b)
    candidatos.add(raiz + spur)
  A.push( menor candidato )
return escolher(curto, médio, longo)
Pilar 4 — Mais algoritmos

Conceitos, pool e a obra mais distante

Grafo de conceitos

Conta a co-ocorrência de pares de atributos no acervo; mantém arestas acima de um limiar e poda top-3 por nó (anti-novelo). Nó = conceito, tamanho = frequência.

Pool (média)

Dado um conjunto S de obras, ranqueia o resto por similaridade média a S: score(x)=avg p∈S sim(x,p).

Obra mais distante

argmin da similaridade (a menos relacionada) que ainda seja alcançável, e então o caminho até ela via Yen.

Complexidades: co-ocorrência O(V·t²) (t = atributos por obra) · ranking/argmin O(V²) · Yen ≈ K execuções de Dijkstra.

Implementação

Visualização & stack

  • Force-directed layout (d3-force): repulsão, molas nas arestas (rigidez ∝ similaridade), colisão e gravidade ajustável.
  • Vue 3 + Vite — interface reativa; render do grafo em SVG.
  • Dados 100% locais — roda offline, sem API.
  • Busca, favoritos, lista e histórico (localStorage).
  • Responsivo — no celular, segurar um nó o joga na pool.
  • No ar 24/7: systemd + túnel Cloudflare em VPS.
